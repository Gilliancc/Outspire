# Outspire 产品需求文档（MVP版）

- 文档版本：V1.0
- 最后更新：2026-06-25
- 产品暂定名：Outspire
- 核心方向：游戏化轻量城市探索工具，抽卡式获取灵感，零负担无强制交互

---

## 一、产品核心定位与设计原则

### 1.1 核心定位

一款「点开即玩、用完即走」的轻量城市探索灵感工具，以扭蛋抽卡为核心交互，为用户提供随机、小众、无目的的城市出行灵感，解决当代人班后/周末「不知道去哪、不想做攻略、反感打卡负担」的痛点，最终实现「重新建立人与日常城市的温柔联结」的内核。

区别于大众点评、攻略类APP的功利性打卡导向，本产品主打**低使用成本、无心理压力、轻度游戏化惊喜**，不做长期行为绑架，靠随机新鲜感与轻仪式感实现自然留存。

### 1.2 底层设计原则（所有功能决策均需遵守）

1. **主交互绝对极简**：首页核心操作永远只有「点一下扭蛋」，所有附加功能全部藏入次级入口，无红点、无弹窗、无强制引导。
2. **零任务零强制**：无签到、无打卡任务、无排行榜、无进度条、无补签机制，用户使用无任何心理负担，断用无负罪感。
3. **软留存轻积累**：留存靠随机惊喜感与自愿式沉淀，所有记录与存档均为用户主动触发，不自动生成待办、不推送提醒。
4. **弱社交无压力**：社交仅做匿名情绪共鸣，无关注、无点赞、无回复、无个人主页，避免社交攀比与互动压力。

### 1.3 目标用户

18-35岁城市年轻群体，日常活动半径固化，周末/班后有闲散时间，反感网红打卡与复杂攻略，偏爱松弛、小众、有新鲜感的轻量出行体验。

## 二、核心功能与规则详情

### 2.1 首页·城市扭蛋机（核心主模块）

首页为产品唯一主界面，视觉核心为动森手绘风扭蛋机，无其他冗余信息。

1. **核心交互**：页面仅1个「扭一下」主按钮，点击触发扭蛋掉落动画，弹出对应灵感卡片，全程操作路径≤2步。
2. **卡池分级与稀有度规则**

   卡片分为3个等级，抽前不公示卡池、不显示概率，完全以随机惊喜感为核心：

   - **普通款**（占比约70%）：日常通用型探索灵感，覆盖短途散步、小众小店、城市角落等常规内容。
   - **稀有款**（占比约25%）：更小众、更具体验感的探索灵感，如冷门观景位、本地老巷、特色通勤路线等。
   - **限定款**（占比约5%）：仅在对应时间、天气、季节条件下才会进入卡池，如下雨天限定、日落时段限定、秋日限定等，错过即无法抽取。

3. **环境自适应规则**

   接入免费天气与时间接口，后台自动根据用户实时的「天气、时段、季节」过滤卡池内容，用户无需手动选择：

   - 雨天自动过滤长时户外灵感，优先匹配室内、短程雨景体验；
   - 深夜自动过滤人流密集点位，优先匹配安静、安全的夜间场景；
   - 不同季节自动匹配当季专属灵感（如春季赏花、秋季扫街）。

4. **抽取上限规则**

   每日单用户最多可抽取**10次**，抽满后按钮置灰，提示「今日灵感够多啦，出门走走吧」。不做付费解锁次数，引导用户回归线下体验，避免刷腻。

### 2.2 今日手气·临时抽屉

满足用户「体验后回溯标记」的需求，全程低存在感，不干扰主流程。

1. **入口规则**：位于扭蛋机下方，以低饱和度小字按钮呈现，无红点、无数字角标、无高亮引导，仅用户主动查找时可见。
2. **展示规则**：收纳当日用户所有抽到过的卡片，按首次抽取顺序排列，**同一张卡片当日重复抽取仅展示1张（去重）**。
3. **清空规则**：每日0点静默清空抽屉内所有卡片，无清空提示、无挽留弹窗、不自动归档。未被主动标记「去过啦」的卡片直接清除，不留任何痕迹。
4. **重复已标记卡片处理**：当日抽到已被用户标记过的卡片，卡片缩略图带"已探索"标记，不影响抽卡主流程体验。

### 2.3 灵感小盒·探索档案

用户自愿沉淀的个人探索记录，无进度统计、无收集压力。

1. **归档规则**：**仅用户主动点击「去过啦」的卡片，才会永久进入灵感小盒**，未标记的卡片永远不会出现在小盒中。
2. **展示规则**：外层为去重卡片墙，同一条灵感永远只展示1张封面卡片，无收集数量统计、无完成度百分比、无分类筛选强制引导。
3. **单卡详情·个人时间轴**

   点击单张卡片进入详情页，内部承载个人探索记录：

   - 记录用户每一次标记「去过啦」的时间，附带当日天气标签，按时间顺序排列；
   - 支持用户每次标记时留下个人碎碎念，对应挂载在对应时间记录下；
   - 完美适配「同个地点、不同时间体验不同」的场景，外层清爽、内层有记忆厚度。

### 2.4 弱社交·城市碎碎念

匿名城市漂流瓶式轻社交，仅做情绪共鸣，无互动压力。

1. **发布交互**

   - 用户点击「去过啦」完成标记后，弹出极轻的可选输入框，提示语为「留一句碎碎念给后来的人？（可不填）」；
   - 输入为纯文本，默认空白，直接点击确认即可完成标记，无需强制输入；
   - 发布全程匿名，无头像、无昵称、无个人主页关联。

2. **展示规则**

   - 位于单张卡片详情页底部，标题为「其他人留下的碎碎念」；
   - 每次打开详情页**随机抽取最多10条**展示，不按时间排序、不显示发布时间、无点赞/回复/举报入口；
   - 每次刷新都可能看到不同的留言，强化「随机偶遇陌生人生活碎片」的氛围感，避免刷评论式沉浸。

3. **边界规则**：MVP阶段仅支持纯文本，**不开放图片上传功能**，降低内容审核成本与用户使用负担。

## 三、核心用户交互流程

### 3.1 主流程：获取探索灵感（核心高频场景）

```
打开APP → 点击「扭一下」 → 查看弹出的灵感卡片 → 直接退出APP，线下体验
```

全程核心操作1步，无多余环节，满足「不知道去哪，点开抽一下就走」的需求。

### 3.2 分支流程：标记到访与留言

```
线下体验完成后 → 打开APP → 点击「今日手气」找到对应卡片
 → 进入详情页点击「去过啦」 → 可选输入碎碎念 → 确认完成
```

### 3.3 回溯流程：查看过往探索

```
打开APP → 进入「灵感小盒」 → 点击任意卡片
 → 查看个人探索时间轴 + 陌生人碎碎念 → 退出结束
```

无强制回访引导，纯用户主动发起。

## 四、MVP版本范围界定

### 4.1 包含功能

1. 首页扭蛋抽卡核心模块（含三级稀有度、天气/时段自适应、每日抽取上限）
2. 「今日手气」临时抽屉（当日去重展示、0点静默清空）
3. 「灵感小盒」探索档案（主动标记归档、去重卡片墙、单卡个人时间轴）
4. 匿名城市碎碎念（纯文本发布、随机10条展示）
5. 动森风全套视觉与动效设计

### 4.2 明确不包含（后续迭代可选）

1. 每日探索小签、主题卡池切换等轻仪式感功能
2. 卡片暂存功能
3. 图片上传功能
4. 用户账号体系、好友社交、排行榜
5. 精准导航、路线规划功能
6. 推送通知、红点提醒机制

## 五、核心观测指标（轻量导向）

1. 核心转化：每日人均抽卡次数、「去过啦」标记率
2. 自然留存：次日自然留存、7日自然留存（无任何运营干预下的用户主动回流）
3. 深度行为：灵感小盒人均卡片数、单卡多次标记占比、碎碎念发布率
